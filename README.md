# QA Bug Report Samples (Exploratory Testing – SaaS Applications)

This repository contains structured examples of manual exploratory bug reports created during testing of modern SaaS web applications.

## Example Bug Reports

- BR-005 – Assignee Name Sync Across Sessions  
  → Cross-session data consistency issue discovered during SaaS workflow testing  

- BR-004 – Task Deletion Without Notification  
  → Missing system feedback and notification gap  

- BR-003 – Settings Modal UX Limitation  
  → UI constraint impacting usability

## Scope

- End-to-end user flow validation
- Multi-role behavior testing
- UX consistency checks
- Cross-platform observations (web / desktop)
- Structured documentation for developer handoff

## Reporting Structure

Each bug report includes:

- Environment
- Preconditions
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity & Type
- Notes / Hypothesis (when relevant)
- Visual Evidence

## Focus

The goal of these reports is not only to identify defects, but to uncover product and workflow risks that may impact real user behavior.
