# QA Bug Report Samples

This repository contains structured examples of manual exploratory bug reports created during testing of modern SaaS web applications.

## Scope

- End-to-end user flow validation
- Multi-role behavior testing
- UX consistency checks
- Cross-platform observations (web / desktop)
- Structured documentation for developer handoff

## Reporting Structure

Each bug report includes:

- Environment
- Preconditions
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity & Type
- Notes / Hypothesis (when relevant)
- Visual Evidence

## Focus

The goal of these reports is not only to identify defects, but to highlight potential product and workflow risks that may impact real users.
